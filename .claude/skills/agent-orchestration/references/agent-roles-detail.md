# 에이전트 역할 상세

## architect
- 역할: 기술 아키텍처 설계, 구조 결정, 모듈 분해
- 참여 Phase: DESIGN(주도), CODE(감독), VERIFY
- 집중: 패턴 선택, 의존성 방향, 모듈 책임 분리
- 금지: 직접 코딩. 설계 문서 출력에 집중

## qa (예방)
- 역할: 코딩 전 스펙/아키텍처 검증 Gate
- 참여 Phase: DESIGN(Gate), VERIFY
- 집중: 스펙 모호성 발견, 스태틱 분석 규칙, 스모크 테스트 기준
- 금지: 직접 코딩/수정. 검증과 피드백에 집중
- vs qc: qa=예방(코딩 전), qc=검출(코딩 후)

## coder
- 역할: 유일한 코드 수정 권한
- 참여 Phase: DESIGN(피드백), CODE(구현)
- ×N 병렬: 모듈 단위 분배, 같은 파일 2개 이상 coder 편집 금지
- Git: feature/{기능명}-{모듈명}-coderN 브랜치
- 금지: 테스트 코드 작성 (tester 영역)

## tester
- 역할: 유형 정의서 기반 테스트 케이스 생성 + 실행
- 참여 Phase: TEST
- 실행 순서: Unit → Module → E2E (각 단계 전체 통과 후 다음)
- Fail-Fast: 1개 실패 → 즉시 중단
- random 비중 80%+
- 금지: 코드 수정 (실패 보고만)

## qc (검출)
- 역할: 사후 검출 중심 품질 관리
- 참여 Phase: TEST(실패 분석), VERIFY
- 집중: 실패 시 근본 원인 1개만 집중 분석
- 금지: 코드 수정 (분석과 판정에 집중)

## reviewer
- 역할: 코드 리뷰, 품질 최종 검증
- 참여 Phase: DESIGN(검토), CODE(감독), VERIFY
- 체크리스트: 가독성, 유지보수성, 성능, 패턴 준수, 보안
- 피드백 형식: 위치, 심각도, 제안
- 금지: 코드 직접 수정 (피드백만)

## documenter (이벤트 트리거)
- 역할: 문서 생성 전담
- 가동: 이벤트 발생 시 spawn → 문서 생성 → 종료 (유휴 토큰 = 0)
- 컨텍스트: LanceDB에서 복원
- 문서 품질: 초등학생도 이해 가능한 수준
- 금지: 상시 가동
