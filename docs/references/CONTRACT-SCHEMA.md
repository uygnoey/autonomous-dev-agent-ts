# Contract 기반 HandoffPackage

## 필수 원칙 5가지

1. **기능 명세**: 무엇을 만드는가 (기능명, 설명, 범위)
2. **아키텍처 설계**: 어떤 구조로 만드는가 (모듈, 인터페이스, 의존성)
3. **테스트 유형 정의**: 어떻게 검증하는가 (카테고리, 규칙, 비율, 샘플)
4. **구현 순서**: 어떤 순서로 만드는가 (파일별 우선순위)
5. **성공 기준**: 언제 완료인가 (수치 기준)

## HandoffPackage 생성 프로세스

1. 1계층에서 유저 확정 후 생성
2. 구조 검증 (5원칙 충족 여부)
3. 정합성 검증 (아키텍처 ↔ 테스트 ↔ 구현순서 일관성)
4. qa가 최종 Gate 검증
5. 2계층 handoff-receiver가 수신 + 재검증

## 검증 실패 시

- 구조 검증 실패 → 1계층에 재요청
- 정합성 검증 실패 → architect가 설계 수정
- qa Gate 실패 → 해당 항목 피드백 후 재검증
